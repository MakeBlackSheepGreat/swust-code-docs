# 国际化 (i18n)

SWUST Code 的 TUI 已接入国际化上下文。当前代码里完整落地的是英文 `en` 和简体中文 `zh` 字典；其余 locale 已在类型、标签和规范化逻辑中预留，后续需要补充字典文件。

## 支持的语言

| 代码 | 语言 | 代码 | 语言 |
|------|------|------|------|
| en | English | zh | 简体中文 |
| zht | 繁體中文 | ja | 日本語 |
| ko | 한국어 | de | Deutsch |
| es | Español | fr | Français |
| ru | Русский | pt | Português |
| ar | العربية | da | Dansk |
| pl | Polski | no | Norsk |
| th | ไทย | tr | Türkçe |

## 语言选择

### 方式 1：环境变量

```bash
export SWUST_CODE_LOCALE=zh
swust-code
```

### 方式 2：自动检测

默认设置为 `auto`，系统会根据以下顺序自动检测：
1. **时区** — 根据系统时区推断语言
2. **环境变量** — 检查 `LC_ALL`、`LC_MESSAGES`、`LANG`
3. **Intl API** — 使用浏览器/运行时的 locale 信息

当前本地实现会同步检测系统 locale，并在中文或繁中环境中合并 `en + zh` 字典；`setLocale()` 会写入 TUI KV 偏好，但 TUI 内可视化语言切换入口还没有完整接入。

## 添加新语言

1. 在 `packages/tui/src/i18n/` 下创建新的翻译文件（如 `ja.ts`）
2. 从 `en.ts` 导入 `Keys` 类型
3. 使用 `satisfies Partial<Record<Keys, string>>` 确保类型安全
4. 在 `locales.ts` 中注册新的 Locale 类型和标签
5. 在 `context/language.tsx` 的字典选择逻辑中合并新字典
6. 如果后续改为动态加载，需要确保首屏命令面板不会先回退到英文

## 翻译字典结构

```typescript
// en.ts - 英文字典（canonical keys）
const en = {
  "tui.prompt.placeholder": "Type a message...",
  "tui.command.memory.title": "Memory",
  "tui.tip.1": "Use /memory to search...",
} as const

// zh.ts - 中文字典
import type { Dictionary } from "./en"
const zh: Dictionary = {
  "tui.prompt.placeholder": "输入消息...",
  "tui.command.memory.title": "记忆",
}
```

缺失的 key 会自动回退到英文。
