# SWUST 优势

龙山灵码基于 MiMo-Code 的长任务运行时，并在中文终端体验、工程防护、项目记忆组织和子智能体配置上增加 SWUST 层。

MiMo-Code 提供记忆、checkpoint、subagent、`goal`、`compose`、Dream / Distill、MCP、LSP 和插件系统。SWUST 层围绕日常终端使用和真实仓库风险补充能力。

## 中文终端体验

终端工具的可用性很大程度取决于信息是否容易扫读。龙山灵码优先处理中文用户每天会看到的内容：

- 命令说明、提示和常用对话框使用中文表达
- 侧边栏展示工作目录、Goal、Task、Todo、LSP、MCP、变更文件、token、费用和缓存状态
- getting started 提示直接指向常用入口
- 主题以深蓝色为主，和龙山灵码当前 TUI 保持一致

这些界面调整用于减少终端里反复查找状态和命令的时间。

## 长任务上下文

真实仓库任务经常跨很多轮对话。只依赖聊天历史，很容易遇到上下文过长、早期约束被稀释、恢复时重新解释项目的问题。

v0.6.0 使用 MiMo-Code 的记忆和 checkpoint 机制，并补充 SWUST 的记忆组织方式：

- `MEMORY.md` 保存项目级长期规则
- facts 目录按 one-fact-per-file 保存项目事实
- `@path` 导入把大型记忆拆成多个文件
- `/memory` 用于检索长期项目知识
- checkpoint 在会话压力增大时记录恢复所需状态

这些机制让下一次会话可以接着项目状态工作。

## 工程防护

AI 编程工具在真实仓库里会直接执行 shell、读写文件和修改文档。错误的命令、错误的写入路径或过早收尾都会影响后续步骤。

SWUST 层增加了几类防护：

- Task Gate：检查未完成任务，减少过早停止
- Bash Safety：分析危险 shell 命令，例如高风险删除、下载即执行、强制重置
- Write Guard：收紧记忆和项目文件的写入边界
- Document Validation：修改结构化文档时检查结构约束
- cache-stable prefix：减少长任务中上下文前缀变化带来的缓存和噪声问题

这些防护不能替代人工判断，但能减少长任务中常见的自动化事故。

## 子智能体配置

MiMo-Code 已经提供 subagent 运行时。龙山灵码增加了项目级配置入口：

```text
/subagent
/subagents
```

在项目内可以为可见子智能体设置：

- 模型
- 思考强度 / variant
- 最大执行步数
- 清除项目覆盖值

这让不同角色可以使用不同策略。例如，调查子智能体使用更快的模型，评审子智能体使用更强的推理设置，验证子智能体限制步数避免无意义展开。

## 适合的场景

适合的任务包括：

- 修复一组关联错误
- 分阶段完成重构或迁移
- 让子智能体分别做调查、实现、评审和验证
- 把项目规则、排障结论和重复流程保留下来

如果任务只需要几句话解释，用普通对话工具会更快。
